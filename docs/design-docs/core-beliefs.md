# Core Beliefs

1. Repository knowledge is the system of record.
2. Progressive disclosure beats monolithic instructions.
3. Agents need discoverable structure more than long prose.
4. Plans are first-class artifacts, not ephemeral chat output.
5. Mechanical enforcement of structure reduces drift.
6. Boundaries should be strict; implementation can be flexible inside boundaries.
7. Favor legibility for future agent runs.
8. Keep product scope tight to ship quickly.
